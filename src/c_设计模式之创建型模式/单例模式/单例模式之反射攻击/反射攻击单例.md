## 反射攻击
### 引入
```
反射攻击是通过反射来修改权限并获取私有构造器, 从而
创建对象的, 进而破坏了单例模式, 下面我们以代码的形式
来看看反射的破坏
```
### 单例模式的类(以饿汉模式为例)
```
public class HungrySingleton {
	private final static HungrySingleton hungrySingleton;
	private HungrySingleton () {}

	static {
		hungrySingleton = new HungrySingleton();
	}

	public static HungrySingleton getInstance () {
		return hungrySingleton;
	}
}
```
### 测试代码
```
public class TestClass {
	public static void main (String[] args) throws ClassNotFoundException, IllegalAccessException,    
            InstantiationException, NoSuchMethodException, InvocationTargetException {
              
    // 正常的单例模式获取对象
	  HungrySingleton instance = HungrySingleton.getInstance();

	  // 对HungrySingleton进行反射
		Class cl = Class.forName( instance.getClass().getName() );
		Constructor constructor = cl.getDeclaredConstructor();
		constructor.setAccessible( true );
		HungrySingleton newInstance = (HungrySingleton) constructor.newInstance();

		// 比较
		System.out.println( instance );
		System.out.println( newInstance );
		System.out.println( instance == newInstance );
	}
}
```

### 分析
```
根据反射可以获取私有构造器这个特性是可以来破坏单例模式的, 那么是否
有办法避免呢? 那就是在私有构造器中进行判断, 如果对象的值不为空则
抛出异常, 即如下代码
```

### 反射防范
```
public class HungrySingleton {
	private final static HungrySingleton hungrySingleton;
	private HungrySingleton () {
		if ( hungrySingleton != null ) {
			throw new RuntimeException( "单例模式不允许增加反射攻击" );
		}
	}

	static {
		hungrySingleton = new HungrySingleton();
	}

	public static HungrySingleton getInstance () {
		return hungrySingleton;
	}
}
```

### 再来看看这种不是在类加载时就创建好了对象的情况
```
public class HungrySingleton {
	private HungrySingleton () {
		if ( hungrySingleton != null ) {
			throw new RuntimeException( "单例模式不允许增加反射攻击" );
		}
	}
	private static HungrySingleton hungrySingleton;

	public static HungrySingleton getInstance () {
		if ( hungrySingleton == null ) {
			synchronized ( HungrySingleton.class ) {
				if ( hungrySingleton == null ) {
					hungrySingleton = new HungrySingleton();
				}
			}
		}

		return hungrySingleton;
	}
}
```
### 分析
```
当我们采用饿汉式或者静态内部类的形式创建的懒汉模式的时候, 会发现
因为在类的加载时就已经存在了对象, 那么之后即使再通过反射去获取
私有构造器来构造对象都会抛出异常, 因为反射的使用也会触发类的加载,
如果是采用两重检验或者同步代码块的懒汉模式, 那么其任何一次的创建对象
都是在私有构造器中创建的, 这时就会出现问题, 如果存在多个线程, 一个线程
是反射的, 一个线程是非反射的, 两个线程同时进入私有构造器, 此时会发现
hungrySingleton是null, 那么就都会触发创建对象, 从而也破坏了单例模式,
目前来说, 这种方法貌似无解=,=只能说我们在使用的时候尽量避免,多注意了
```
