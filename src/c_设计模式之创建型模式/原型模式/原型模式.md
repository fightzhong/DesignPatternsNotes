## 原型模式
### 引入一个场景
  ```
  假设学校要给所有的同学们发送一封邮件, 邮件对象
  为Mail类, 通过MailUtil里面的send方法发送(假设),
  那么我们看看下面的代码
  ```
### 未利用原型模式的邮件发送
```
邮件类:
  public class Mail {
    private String name;
    private String address;
    private String content;

    public Mail () {}

    setter && getter ....
  }

MailUtils类
  public static void sendMail (Mail mail) {
		String mailContent = "收件人: {0}, 收件地址: {1}, 邮件内容: {2}";
		System.out.println( MessageFormat.format( mailContent, mail.getName(),
									mail.getAddress(), mail.getContent() ) );
	}

	public static String saveOriginMail (Mail mail) {
		return mail.getContent();
	}

测试类:
  public static void main (String[] args) {
    for ( int i = 0; i < 10; i ++ ) {
      Mail mail = new Mail();
      mail.setName( "姓名" + i );
      mail.setAddress( "姓名" + i + "@qq.com" );
      mail.setContent( "恭喜你, 收到了邮件..." );
      MailUtils.sendMail( mail );
    }
  }   
```

### 分析
```
我们可以看到, 在测试类中, 我们如果要给10个人发送邮件, 就要
创建10个Mail对象, 如果创建一个对象需要很复杂的操作的话, 那么
就会消耗大量的性能, 而通过原型模式就能够在一定程度上缓解这个
情况, 原型模式通过clone克隆的方式来直接复制二进制流, 从而使得
对象的创建很快, 需要注意的是, 通过克隆的对象是不会执行构造方法
的。
```
### 改进类
```
邮件类:
  public class Mail implements Cloneable {
    private String name;
    private String address;
    private String content;
    public Mail () {}

    // 重写克隆方法, 使得其能在本包内调用
    protected Object clone () throws CloneNotSupportedException {
      return super.clone();
    }

    setter && getter ....
  }

MailUtils类(没变)
  public static void sendMail (Mail mail) {
    String mailContent = "收件人: {0}, 收件地址: {1}, 邮件内容: {2}";
    System.out.println( MessageFormat.format( mailContent, mail.getName(),
                  mail.getAddress(), mail.getContent() ) );
  }

  public static String saveOriginMail (Mail mail) {
  return mail.getContent();
	} 

测试类
  public static void main (String[] args) {
		Mail mail = new Mail();
		for ( int i = 0; i < 10; i ++ ) {
			Mail temp = (Mail) mail.clone();
			temp.setName( "姓名" + i );
			temp.setAddress( "姓名" + i + "@qq.com" );
			temp.setContent( "恭喜你, 收到了邮件..." );
			MailUtils.sendMail( temp );
		}
	}
```

### 分析
```
我们通过给Mail类实现了Cloneable接口, 并重写了clone方法
(因为clone方法是Object的, 并且是protected的, 会在测试类
所在的包下不能被调用), 使得Mail类可被克隆, 之后for循环中
不再进行new, 而是进行克隆, 加快了类的创建
```

### 注意
```
<1> 克隆的对象没有执行构造函数
<2> 克隆是浅克隆, 如果对象中成员属性是非基本类型, 则会使得
    克隆后的对象和原对象中的成员属性指向同一个非基本类型
```

### 总结
```
定义: 指原型实例指定创建对象的种类, 并且通过拷贝这些原型创建新的对象
特点: 不需要知道任何创建的细节, 不调用构造函数
适用场景: 
    <1> 类初始化消耗较多资源
    <2> new产生的一个对象需要非常繁琐的过程(数据准备、访问权限等)
    <3> 构造函数比较复杂
    <4> 循环体中产生大量对象时

优点:
  <1> 原型模式性能比直接new一个对象性能高
  <2> 简化创建过程

缺点: 
  <1> 必须配备克隆方法
  <2> 对克隆复杂对象或对克隆出的对象进行复杂改造时, 容易引入风险
  <3> 深拷贝、浅拷贝要运用得当
```
