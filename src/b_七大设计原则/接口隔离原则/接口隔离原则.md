## 接口隔离原则
### 引入
    ```
    以之前说的单一职责原则进行对比, 单一职责原则针对的是类、
    接口、方法, 而接口隔离原则跟单一职责原则中针对的接口是
    类似的, 就是接口的职责应该单一
    
    定义: 用多个专门的接口, 而不使用单一的总接口,客户端不应该
          依赖它不需要的接口
    
    一个类对一个类的依赖应该建立在最小的接口上, 接口的方法尽量少,
    但是一定要适度
    
    优点: 符合我们常说的高内聚低耦合的设计思想, 从而使得类具有很好
          的可读性、可扩展性和可维护性
          
    如下所示, 人这个接口可以提供很多的技能, 吃得快,跑得快, 游得快,
    根据接口隔离原则, 客户端不应该依赖它不需要的接口, 所以进行了解耦,
    将人这个接口分成了三个接口            
    ```
### 原接口
    ```
    public interface People {
        void runFastSkill ();
        void swimFastSkill ();
        void eatFastSkill ();
    }
    ```  
### 解耦
    ```
    public inteface runFastPeople { void runFastSkill (); }
    public inteface swimFastPeople { void swimFastSkill (); }
    public inteface eatFastPeople { void eatFastSkill (); }
    ```
