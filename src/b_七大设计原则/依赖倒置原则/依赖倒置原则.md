## 依赖倒置原则

### 引入
    ```
    假设一个人需要吃苹果, 那么只需要给她一个苹果, 他就会自己吃了,如:
    public class People {
    	public void eatFruit (Apple apple) {
    		System.out.println( "人在吃" + apple.getName() );
    	}
    }
    
    public class Apple {
    	public String getName () {
    		return "苹果";
    	}
    }
    ```
### 存在的问题
    ```
    可以看到人的吃水果的方法中是传入了一个Apple类, 那么一旦以后
    人想要吃香蕉, 梨子了就会发现吃不了, 从而发现苹果和人两个类
    耦合度很高
    ```    
### 优化
    ```
    public class People {
    	public void eatFruit (Fruit fruit) {
    		System.out.println( "人在吃" + fruit.getName() );
    	}
    }
    
    public interface Fruit {
    	String getName ();
    }
    
    public class Apple implements Fruit {
    	public String getName () {
    		return "苹果";
    	}
    }
    
    public class Banana implements Fruit {
    	@Override
    	public String getName () {
    		return "香蕉";
    	}
    }

    
    // 应用层的调用
    public class TestClass {
        @Test
        public void test () {
            People people = new People();
            // 吃苹果
            people.eatFruit( new Apple() );
            // 吃梨
            people.eatFruit( new Banana() );
        }
    }
    ```
### 分析
    ```
    通过以上的代码可以看到, 我们将人这个类进行了优化, 参数
    由一个Apple改为了Fruit, Apple实现了Fruit, 这叫做面向
    接口编程, 不针对具体的类, 从而以后人可以吃各种水果, 只需要
    这个水果继承自Fruit即可, 并且我们可以不用去修改People的代码
    ```    
    
### 依赖倒置原则
```
定义: 高层模块不应该依赖底层模块, 二者都应该依赖其抽象,
      抽象不应该依赖细节, 细节应该依赖抽象
      针对接口编程
优点: 可以减少类间的耦合性、提高系统稳定性, 提高代码可读性和
      可维护性, 可降低修改程序所造成的风险
              
```
