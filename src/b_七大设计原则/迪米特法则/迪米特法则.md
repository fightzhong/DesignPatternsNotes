## 迪米特法则
> 学习该法则看过许多的博客, 终于在一篇知乎上找到一个相对明白的解释了...

### 一个违反了迪米特法则的例子
    ```
    假设一个超市有收银员负责收钱, 顾客买了东西之后需要跟收银员结算, 一共
    有三个类, 一个是收银员, 一个是顾客, 还有一个是钱包, 顾客需要从钱包
    中拿出钱来支付收银员计算出的价格, 如下代码所示:
    
    钱包对象:
        public class Wallet {
            double money;
        
            // 获取钱包里有多少钱
            public double getMoney () {
                return money;
            }

            // 从钱包里提取钱
            public void subtractMoney (double money) {
                this.money -= money;
            }
        }
    
    顾客对象:
        public class Customer {
            String name;
            Wallet wallet;
        
            // 拿出钱包
            public Wallet getWallet () {
                return wallet;
            }
        }
    
    收银员对象:
        public class Cashier {
            // 收钱
            public void pay (Customer customer) {
                Wallet wallet = customer.getWallet();
                wallet.subtractMoney( 100 );
            }
        }
    ```
### 分析
```
以上的代码可以这样认为, 收银员告诉顾客, 拿出你的钱包, 顾客老老实实的拿出
钱包交给了收银员, 收银员直接从钱包里拿钱......是不是觉得很惊奇, 这就是违
反了迪米特法则, 也违背了人们的认知, 按照正常的情况来说, 这个拿钱的动作应该
由顾客来完成, 所以我们的代码需要进行以下优化.
```
### 优化代码
    ````
    顾客对象:
        public class Customer {
            String name;
            Wallet wallet;
        
            // 拿出钱包
            public Wallet getWallet () {
                return wallet;
            }
            
            // 顾客付钱
            public void payMoney (double money) {
                wallet.subtractMoney( money );
            }
        }
    
    收银员对象:
        public class Cashier {
            // 收钱
            public void pay (Customer customer) {
                customer.payMoney( 100 );
            }
        }
    ````

### 迪米特法则分析
    ```
    定义: 迪米特法则又叫做最少知识原则, 就是说一个对象应当对其它对象有尽可能
          少的了解, 不和陌生人说话
    朋友: 迪米特法则中会提到一个朋友的概念, 朋友指的是一个类中的成员变量, 成员
          方法中出现的类, 例如在收银员类中, 收银员和顾客是朋友, 而收银员和钱包
          不是朋友, 那么既然不是朋友就不能直接调用钱包的方法, 否则就是违反了
          迪米特法则   
    所以我们通过改进这个类的设计, 将钱包付钱的动作转移到了顾客的手中, 从而实现了
    收银员和钱包的解耦. 
    
    迪米特法则规定, 如果两者不是朋友, 那么就不同调用非朋友的方法, 即不能出现类似
    这样的链式调用m1.m2().m3().m4()......但是思考下面的情况:
    
    str.split( "&" )
        .stream()
        .map( str -> str.contains(elementName) ) ? str : ""
        .filter( str -> !str.isEmpty() )
        
    这上面虽然出现了类似的链式调用, 但是会发现这个调用返回的是自身对象, 而不是别的对象,
    我们在收银员的例子中调用getWallet对象返回的是Wallet对象, 所以是违反了迪米特法则的    
    ```
